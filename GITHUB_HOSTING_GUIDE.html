Step-by-Step GitHub Pages Hosting & PDF Conversion Guide

💡 Quick Tip: Click the Print or Save as PDF Guide button to save this hosting guide locally as a clean, styled PDF! Use the Export to ZIP option in the Google AI Studio menu to download the clean compiler-optimized files.

1. Export Your Code from Google AI Studio

  1. In the Google AI Studio environment, look at the top-right toolbar.
  2. Click the Settings cog or the export button in the sidebar.
  3. Select Export Code as ZIP to package your workspace.
  4. Once downloaded, extract the ZIP file into a folder on your machine (e.g. kisan-website).

2. Create a Free Repository on GitHub

  1. Log in to github.com.
  2. Click the New button (or + in the header) to create a new repository.
  3. Name your repository kisan-shuttering.
  4. Set it to Public (required for free web hosting).
  5. Leave all default initialization boxes unchecked (No README, .gitignore, etc., as they are pre-configured in your code).
  6. Click Create repository.

3. Automated Deployment to GitHub Pages (Actions)

Because your website uses Vite & Tailwind, it needs to be compiled before it runs in production. I have already fully configured a premium, automated CI/CD pipeline for you! The folder .github/workflows/deploy.yml has already been created in your project with the following contents:

name: Deploy Kisan Shuttering Website to GitHub Pages

on:
  push:
    branches:
      - main
      - master

permissions:
  contents: write
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  build-and-deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install Dependencies
        run: npm install

      - name: Build Production Bundle
        run: npm run build

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload Build Artifacts
        uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

4. Upload and Push

Open your local terminal in the extracted folder folder, and run these commands to linked files to GitHub:

# Initialize git
git init

# Add all project source files
git add .

# Save snapshot
git commit -m "Launch Kisan Web Application"

# Rename local branch
git branch -M main

# Link to your new GitHub repository
git remote add origin https://github.com/YOUR_USERNAME/kisan-shuttering.git

# Push repository files (Replace YOUR_USERNAME)
git push -u origin main --force

5. Select Delivery Source

  1. Go to your GitHub online repository.
  2. Click the Settings tab and open Pages in the sidebar.
  3. In Build and Deployment -> Source, change the dropdown selection to GitHub Actions.
  4. Your site will now build in parallel and deploy to your custom public link: https://YOUR_USERNAME.github.io/kisan-shuttering/
